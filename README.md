# 工包管理小工具
这是通过AI coding 生成的一个极简工包管理小工具，html文件可直接以浏览器形式打开。

如果你们公司是内网环境，可以使用offline版本，把html和xlsx.full.min.js文件一起下载到同一文件夹。

如果你们公司内可以联网，可以使用online版本，只需下载html文件即可。

## Why? 我为什么要ai coding 这个小工具？

想做这个小工具的初衷是我在工作中会负责工时分配的任务。简言之就是自有员工和外员每周/每天都需要填写工包。主要遇到的困难有以下两点：  
- 工包众多，一年会有多个批次，每个批次会有多个任务，一个任务根据开发、功能测试、投产演练等不同阶段会有多个工包。
- 工包时间交叉，可能第一个批次进行到功能测试阶段，第二个批次在开发阶段，第三个批次在投产演练阶段。

这就导致会出现的以下情况：工包已经过了结束日期，还没有分配，导致工包浪费
- 先分了靠后的工包，导致前期工包浪费，后期没有工包可分
- 或者先分了后一个批次的工包，前期的某些工包浪费。

总之这个工包分配的小工具主要是为了解决以上问题，也就是说能让我每周五分配工包时，知道我最应该先分配哪些工包。

后来又出现了一个小需求，我希望知道，当前任务的工包能供自有人员和外员填写多久？也就是说我会不会有工包富余或者紧缺的情况，如果有紧缺的情况我需要评估还要新增多少工包才可以覆盖。

## What？这个小工具是什么？

三个主要功能：
- **工包汇总**：可以按照人员类型（自有和外员）以及不同批次信息，汇总工作量相关信息
- **可分配工包**：可以看到在当前时间能分配哪些工包，这些工包的结束时间是由近及远排列。
- **工包预警**：根据产能和剩余工包的情况做比较。看是否紧缺
